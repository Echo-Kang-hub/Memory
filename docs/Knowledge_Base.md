# AI Agent 协作与 Prompt 工程知识库

## 1. 核心交互原则

为了确保 Agent 输出的高质量，请遵循以下 **CO-STAR** 框架：

- **C (Context) 上下文**: 提供背景信息，说明任务的原因。
    
- **O (Objective) 目标**: 明确希望 Agent 完成的具体任务。
    
- **S (Style) 风格**: 指定写作风格（如：幽默、专业、简洁）。
    
- **T (Tone) 语气**: 设置情感基调（如：鼓励、严谨、谦虚）。
    
- **A (Audience) 受众**: 明确输出内容是给谁看的。
    
- **R (Response) 响应**: 规定输出格式（如：Markdown, JSON, 表格）。

## 2. 结构化提示词模板

在构建本地 Agent 时，使用 Markdown 结构能显著提高其解析准确率：

Markdown

```
# Role: [角色名称]
## Profile:
- Author: [你的名字]
- Language: 中文
- Description: [简要描述角色的核心能力]

## Goals:
1. [目标1]
2. [目标2]

## Constraints:
- [约束条件1]
- [约束条件2]

## Workflows:
1. 第一步：分析用户输入。
2. 第二步：检索相关知识。
3. 第三步：生成结构化回复。
```

## 3. 常见技术概念简表

如果你的 Agent 涉及技术问答，可参考以下标准定义：

|**概念**|**定义**|**应用场景**|
|---|---|---|
|**RAG**|检索增强生成 (Retrieval-Augmented Generation)|给模型挂载外部“大脑”（文档/数据库）|
|**Temperature**|温度参数|越高越随机/有创意，越低越稳定/严谨|
|**Token**|文本单元|模型处理文本的基本单位（通常 1 个汉字 ≈ 1-2 tokens）|
|**Zero-Shot**|零样本提示|直接下指令，不给任何示例|

## 4. 逻辑推理增强 (Chain of Thought)

当处理复杂逻辑时，务必在提示词中加入以下指令：

> "请逐步思考并推导你的结论。"

这能激活模型的**思维链 (CoT)**，通过显式地写出推理过程，减少逻辑幻觉。

---

## 5. 错误处理与调试

如果 Agent 输出不符合预期，请检查以下三点：

1. **指令冲突**：是否有两条指令相互矛盾？
    
2. **信息缺失**：是否给了结果要求，但没给必要的输入数据？
    
3. **粒度过粗**：尝试将复杂任务拆分为多个子步骤。
